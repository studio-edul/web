# 403 Forbidden 에러 해결 가이드

## 가능한 원인들

### 1. 데이터베이스에 Integration 연결 확인 (가장 중요!)

각 데이터베이스(CV, WORK, ARTWORK)에 Integration이 연결되어 있는지 확인:

1. Notion에서 각 데이터베이스 페이지 열기
2. 우측 상단 "..." 메뉴 클릭
3. "Connections" 또는 "연결" 선택
4. 생성한 Integration이 체크되어 있는지 확인
5. **각 데이터베이스마다 개별적으로 연결해야 합니다!**

### 2. CORS 프록시 문제

현재 사용 중인 `cors-anywhere.herokuapp.com`은 더 이상 공개적으로 사용할 수 없을 수 있습니다.

**해결 방법:**
- 다른 CORS 프록시 서비스 사용
- 또는 백엔드 서버 구축 (추천)

### 3. 데이터베이스 ID 형식

코드에서 하이픈을 자동으로 제거하도록 수정했습니다. Notion API는 하이픈 없는 형식을 선호합니다.

### 4. 에러 응답 확인

이제 에러 발생 시 더 자세한 정보가 표시됩니다. 에러 메시지를 확인하여 정확한 원인을 파악하세요.

## 확인 체크리스트

- [ ] CV 데이터베이스에 Integration 연결됨
- [ ] WORK 데이터베이스에 Integration 연결됨
- [ ] ARTWORK 데이터베이스에 Integration 연결됨
- [ ] API 키가 올바르게 입력됨
- [ ] 데이터베이스 ID가 올바르게 입력됨

## 테스트 방법

1. `test-notion.html` 파일을 브라우저에서 열기
2. 브라우저 개발자 도구(F12) 열기
3. "WORK 데이터 가져오기" 버튼 클릭
4. 콘솔과 네트워크 탭에서 에러 상세 정보 확인

## 추가 디버깅

브라우저 개발자 도구의 Network 탭에서:
1. 실패한 요청 클릭
2. Headers 탭에서 요청 헤더 확인
3. Response 탭에서 에러 메시지 확인
